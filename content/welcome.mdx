## 使用指南

### 环境设置
1. 创建虚拟环境：
```bash
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# 或
.\.venv\Scripts\activate  # Windows
```

2. 安装依赖：
```bash
pip install -r requirements.txt
```

### 项目规范
1. 每个新的学习主题或实验应该创建独立的目录
2. 使用清晰的命名约定（如：snake_case）
3. 保持代码文档的及时更新
4. 定期整理和归档学习笔记

### Jupyter Notebooks 使用建议
1. 将长期运行的实验放在 `notebooks/experiments/` 目录
2. 将教程和学习笔记放在 `notebooks/tutorials/` 目录
3. 每个 notebook 文件名应该包含日期前缀（如：`20240315_llm_evaluation.ipynb`）

## 依赖管理
- 主要依赖包都列在 `requirements.txt` 中
- 不同实验可能需要不同的依赖，建议在相应目录下维护单独的 requirements 文件

## 注意事项
1. 请勿将敏感信息（如 API 密钥）直接写入代码
2. 大型数据集请不要直接提交到代码库
3. 保持代码库整洁，定期清理不需要的文件

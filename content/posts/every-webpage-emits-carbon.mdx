# 每个网页都会排放二氧化碳

如果你关心环境，请打印这篇文章。

## 一、数字环保主义的兴起与争议

这篇文章最初发表于 2018 年 12 月。当时，《Low Tech Magazine》的"太阳能"版本[^1]树立了新标准，激起了许多人对这个主题的兴趣。从 websitecarbon.com[^2]开始——该网站也于 2018 年底发布，并在 Danny van Kooten 2020 年初关于他流行的 WordPress 插件的帖子[^3]中被提及——以及 Kinsta，后者在 2021 年 1 月发表了一篇关于网站排放的文章[^4]。

但让我们考虑一个引起媒体轰动的例子。2021 年 2 月 8 日，Formafantasma[^5]——一个在阿姆斯特丹工作的意大利设计二人组——更新了他们的网站。这个消息被《Dezeen》[^6]和《Artribune》[^7]等媒体报道，重复了网站本身宣称的说法："设计用于最小化浏览互联网所产生的能源消耗和二氧化碳排放"[^8]。

"界面——介绍继续道——使用系统字体（Arial 和 Times New Roman）以避免不必要的 HTTP 请求，并且默认提供深色模式，遵循操作系统颜色方案的偏好，从而降低屏幕亮度并减少能源消耗——特别是在移动使用中，OLED 屏幕最为常见。"

让我们忘记 OLED 屏幕能源消耗的复杂性，以及亮度设置和 RGB 值本身等因素[^9]：仍然很难想象用户会在 Formafantasma 的网站上花费如此多的时间，以至于他们会在电池方面注意到可量化的积极影响。

抛开系统字体的使用不谈——这个问题我们已经在《Nothing special》中写过——这里重要的是关于减少 HTTP 请求的说法，因为正如许多人在《Dezeen》文章评论中指出的那样[^10]，Formafantasma 的网站并没有兑现这一承诺。

主页发出 11 个请求：除了 HTML 页面外，还有两个 CSS 表、一个 svg 图标（由于一个 bug，每次到达页面底部时都会不断请求），以及七个 JavaScript 文件，包括 jQuery，仅此一项就占 33kB。此外，如果你没有跟踪拦截器，页面会下载 Gtag，它不仅向 Google 出售你的浏览数据，还会再占用 100kB+：基本上是网站本身的两倍。而且主页不包含任何过去项目的图片："作品"页面情况更糟，总共发出 124 个 HTTP 请求，传输了 9MB。请注意，由于图片下载基于滚动位置而不是用户实际打开项目，你只需要滚动到页面底部就能达到那 9MB，甚至无需看到一张图片。

相比之下，《Build in Amsterdam》[^11]的主页（从 Formafantasma 同一城市的网站中随机选择）没有宣扬环保理想主义，实际上似乎将炫酷效果作为其特色，但总共只发出 28 个 HTTP 请求，传输 5MB。

另一个类似的案例是《Organicbasic》[^12]，它自豪地宣称自己是"低影响网站"，并有一个专门的页面来证明他们的承诺是多么具体和真诚。可惜的是，该网站基于 Nuxt.js，这是一个服务器端有数百兆字节 JavaScript 的框架，仅用于他们强调加载"关键"资源重要性的宣言就传输了 1MB 的文件（当然，每个要点都由一个"关键"的 svg 图标突出显示）。

甚至我们上面提到的《Kinsta》文章[^13]也重达 6.6MB，超过 100 个请求。毕竟，参考我们之前关于互联网站点可访问性的文章，这里分析的页面都没有通过 WCAG 测试。我们让读者自己得出结论。

## 二、"绿色清洗"，隐藏在环保主义的外表后面

"打印此邮件前请考虑环境"曾经是任何邮件底部的常见建议，用绿色书写。有不同版本，但许多来自 thinkbeforeprinting.org[^14]，一个 2009 年的网站。"我们不想阻止人们打印"——Ink Factory 说，这家打印机墨盒销售商推广了这项活动。"我们只是认为有时，人们，不是所有人，需要提醒浪费纸张、墨水和碳粉在经济或环境上都没有意义。"

一个无害的信息，就像许多其他抨击纸质文档存在以支持完全数字化操作的信息一样。"节约纸张——拯救树木"，然而互联网也有显著的能源影响。

互联网排放二氧化碳——而且比你想象的要多。在服务器上存储数据、处理数据、响应用户请求、拥有传输数据所需的基础设施、天线、中继器、路由器、信号放大器、CDN[^15]、调制解调器，一直到最终设备查看页面所使用的能源。还有更多：还有设备本身的成本，以及在技术进步的竞赛中不断更新的成本，这受到在数字世界运作的意愿增加的鼓励。

总的来说，根据 Emerge[^16]和 Energuide[^17]的估计，通过互联网传输 1GB 数据会产生 2 到 3 公斤的二氧化碳排放。即使估计值较低，比如每 GB 约 29 克[^18]，考虑到每天创建的艾字节（100 万 GB）数据量，长期结果仍然应该至少与减少汽车燃料消耗的需求一样让公众感到震惊。

但为什么不只是提醒人们不要打印邮件呢？上面提到的签名，以绿色文本的 gif 图像形式提出，重约 3kB。需要 30 万封邮件才能达到 1GB，因此达到释放的二氧化碳估计值：绝对无关紧要，除了每天发送超过 2810 亿封邮件[^19]。如果每封邮件都包含那个无害的小环保信息作为图像，我们将有 80 万 GB 传输最多两次（一次发送，一次阅读），因此至少产生 23 吨二氧化碳。与最终进入垃圾箱的邮件不同，至少纸张可以回收利用。

## 三、每个网页平均重 3MB

"绿色清洗"，隐藏在环保主义的外表后面。在普通用户无法评估的编程层面，声明并未真正反映。虽然完全不使用汽车总是比拥有新的环保汽车更好，但如果这个或其他网站必须存在，我们至少应该怀疑他们的声明是否得到行动的确认。

这个页面的第一个版本发出 10 个 HTTP 请求，总共传输 128kB；最近对网站的大修使文章减少到 4 个请求，总共传输 40kB——基本上只有文本和相关插图。不幸的是，我们提到的其他网站并非如此——也不是 Norma 签署的一些网站：但在最近几年，我们一直注意大幅减少图片和视频的重量，并避免使用 jQuery 等重型 JavaScript 库，尽可能压缩我们实际需要的资源。

事实是，今天，每个网页平均重 3MB[^20]：这个数字在过去十年中几乎增加了四倍。撇开广泛使用高数据传输的网络服务，如电影流媒体、视频通话和云游戏不谈，如果我们考虑普通网页，内容比 2008 年好得多，以至于它们证明其能源滥用是合理的吗？

对具有高视频和图片内容的复杂页面日益增长的兴趣没有帮助，特别是考虑到 Google 不会惩罚而是倾向于包含视频的页面。网络使用的合理化主要取决于设计师和程序员：因此需要他们负起责任，停止设计具有重型背景照片或其他处理器密集型功能的网站，这些功能不会增加内容，更糟糕的是，如果这样做是以虚假环保主义的名义。

虽然我们清楚地知道这些实验和这些话在 Facebook 应用更新的背景下是完全不公平的，该更新迫使数百万用户每周下载近 200MB[^21]，并且为了没有人要求的新功能，我们仍然认为证明每个选择都有有形影响是明智的，即使在数字、虚拟维度中也是如此。
